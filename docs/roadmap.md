# Blank AI 后续开发路线图

这份文档记录当前 MVP 之后的重点待实现事项，避免后续迭代时遗漏关键工程方向。

## 当前阶段判断

Blank AI 当前已经进入 AI 页面编辑器 MVP 阶段：极简输入、AI patch 更新页面、图片生成、动态 React 组件、历史时间轴、撤销/跳转、后端校验和测试基线都已经具备。

但它还不是生产级产品。主要风险集中在生成组件安全边界、AI 工作流稳定性、资源持久化、错误恢复和真实浏览器端测试。

## P0：稳定性与可观测性

- 为每轮用户请求保留完整 workflow trace，包括 planner 输出、工具调用、工具结果摘要、repair 过程和最终 patch。
- 增加可读的 debug 查看入口，便于定位是文本模型、图片模型、patch 校验、代码生成还是前端渲染失败。
- 优化图片生成、代码修复和 provider 超时失败时的用户体验，失败时不要污染历史快照。
- 为常见失败提供 fallback，例如图片失败时先保留组件结构并提示可以重试生成背景图。
- 将日志字段标准化，至少包含 requestId、sessionId、prompt preview、model、tool name、duration、status、error。

验收标准：

- 任意一次失败都能通过日志定位失败层级。
- 模型或图片接口失败时，页面保持可用，历史不出现坏快照。
- 用户可以继续输入让 AI 修复或重试。

## P0：真正的生成组件沙箱

- 将 `generated_react_component` 从主页面 `new Function` 执行迁移到 sandbox iframe。
- iframe 只通过 `postMessage` 接收 theme、mountProps、snapshots 和 activeSnapshotId。
- iframe 不能访问宿主 DOM、cookie、localStorage 或任意全局对象。
- Host SDK 通过能力白名单桥接，例如 `sendPrompt`、`selectSnapshot`、`exportCanvasPng`。
- 未声明 capability 的调用必须被拒绝，并返回可读错误。
- 生成组件运行时报错时，本轮 patch 不污染历史，必要时自动回退。

验收标准：

- AI 生成组件可以正常渲染和交互。
- 组件无法直接访问主页面 DOM 或浏览器敏感状态。
- `sendPrompt`、`selectSnapshot`、`exportCanvasPng` 仍由宿主执行，核心逻辑不可被 AI 改坏。

## P1：会话、快照与资源持久化

- 将 session、messages、snapshots、pageState 从内存迁移到持久化存储。
- 生成图片不再长期以内联 base64 存在 pageState 中，改为 assetId 或 URL 引用。
- 支持刷新页面后恢复当前会话。
- 支持服务重启后继续访问历史快照。
- 为历史快照增加版本字段，方便后续 schema migration。

可选实现方向：

- 本地开发优先使用 SQLite 或文件存储。
- 生产环境可迁移到 Postgres、D1、R2、S3 或其他对象存储。

验收标准：

- 刷新页面不丢当前画布。
- 服务重启后仍能恢复会话和历史点。
- 大图不会让 pageState 和接口响应无限膨胀。

## P1：AI 工作流模块化

- 拆分当前 AI 工作流代码，避免单文件持续膨胀。
- 建议拆分为 providers、planner、tools、patch validation、repair、fallbacks、logging。
- 明确定义工具协议和工具结果格式。
- 将图片生成、图片编辑、组件检查、组件背景 patch 准备等工具独立测试。
- 为 planner/finalizer 增加更严格的结构化输出约束。

验收标准：

- 新增一个 AI 工具时不需要修改大段核心流程。
- provider 替换时不影响 patch 校验和前端协议。
- 每个工具都能单独测试成功、失败和超时路径。

## P1：生成组件质量与一致性

- 为 AI 生成组件增加明确的设计 token、布局规范和可访问性规则。
- 增加组件级 smoke test：至少检查代码可编译、可渲染、基础交互不报错。
- 对常见组件类型建立模板，例如表格、计时器、控制面板、按钮组、卡片、图表占位。
- 支持 AI 对已有组件做定点修改，而不是经常整体重写导致位置漂移。
- 增加组件一致性检查，避免每轮生成出完全不同视觉语言。

验收标准：

- 表格、按钮、计时器等基础组件可稳定生成并可交互。
- 修改组件背景、颜色、位置时不破坏组件原有功能。
- 多轮对话后组件位置和布局不会无故漂移。

## P2：产品 UX 打磨

- 优化生成中的状态显示，避免与时间轴、输入框或生成内容重叠。
- 文本型 AI 回复支持更长内容展示，避免被输入框或页面内容遮挡。
- 增加轻量重试入口，但保持默认界面极简。
- 优化时间轴 hover tooltip 的视觉一致性。
- 为移动端适配输入框、时间轴、生成组件和错误提示。

验收标准：

- 用户在桌面和移动端都能稳定输入、查看生成结果、切换历史。
- 错误提示清楚但不打破极简风格。
- 生成中、文字回复、时间轴和输入框互不遮挡。

## P2：导出能力增强

- 评估当前基于 SVG `foreignObject` 的导出稳定性。
- 对图片、跨域资源、生成组件、特殊 CSS 增加导出兼容测试。
- 必要时引入更可靠的截图方案，例如浏览器端 canvas 渲染或后端 headless browser 截图。
- 支持导出当前快照、指定历史快照或完整时间线。

验收标准：

- 有图片背景和生成组件的页面可以稳定导出。
- 导出的结果与当前画布视觉基本一致。

## P2：浏览器端 E2E 和视觉回归测试

- 增加 Playwright 或 Chrome 自动化测试。
- 覆盖核心链路：启动、输入 prompt、生成图片、创建组件、点击组件、切换时间轴、撤销、导出。
- 增加极简 UI 的视觉回归，避免输入框、时间轴和提示层位置被意外改坏。
- 增加 provider contract 测试，覆盖 MiniMax/Gemini 或后续模型服务的请求路径和解析逻辑。

验收标准：

- 每次重要改动后能自动验证主交互链路。
- UI 关键位置变化可以被测试捕获。

## 暂缓事项

- 多用户账号体系。
- 云端协作编辑。
- 任意 npm 包动态安装。
- 完整自由网页代码生成。
- 复杂拖拽画布和可视化编辑器。
- 语音输入和 TTS。

这些能力可以后续做，但当前优先级低于安全沙箱、稳定性、持久化和生成质量。
